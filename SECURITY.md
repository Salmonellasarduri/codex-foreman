# Security Policy

## Supported versions

Security fixes target the latest GitHub release and the current `main` branch.
Older releases may not receive backports.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use
[GitHub's private vulnerability reporting](https://github.com/Salmonellasarduri/codex-foreman/security/advisories/new).

Include the affected version or commit, shell and operating system, impact,
and a minimal reproduction. Remove credentials, Codex transcripts, private
briefs, and proprietary repository content.

Reports involving shell injection, unsafe path handling, worktree boundary
violations, transcript leakage, or destructive git behavior are especially
important. Ordinary guard or retry bugs can use the public bug report form.
